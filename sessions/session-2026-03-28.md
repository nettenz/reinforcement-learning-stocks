## Context
This session focused on stabilizing dashboard execution, fixing model/data compatibility for Signal Analytics, and substantially upgrading charting + diagnostics so training runs are easier to interpret and compare across environments.

## Achievements in this session

### 1) Dashboard run reliability
- Verified the launcher behavior on macOS and identified why startup appeared successful while app was not actually running.
- Found root cause in runtime log:
  - `.venv/bin/python: No module named streamlit`
- Installed dependencies with project Python and confirmed service was healthy on `http://localhost:8501`.
- Repeatedly validated process lifecycle:
  - `start`, `status`, `stop` flow via `run_dashboard.sh`.

### 2) Fixed observation shape mismatch crash in Signal Analytics
- Error resolved:
  - `Unexpected observation shape (12,) ... expected (7,)`
- Root cause:
  - Saved PPO model expected 7-dim observations while current input data/env could expose additional sentiment features.
- Implemented robust model-env alignment in `src/signal_analytics.py`:
  - Detects expected observation dimension directly from loaded model.
  - Automatically aligns dataframe sentiment columns to required dimension.
  - Validates final environment observation shape before inference.

### 3) Screenshot workflow support
- Added `screenshots/` to `.gitignore` so screenshots can be shared for debugging without polluting git status.

### 4) Signal analytics behavior diagnostics (no Buy/Sell)
- Investigated “empty Buy/Sell tables” and confirmed it was model behavior (not a rendering bug).
- Measured policy behavior:
  - Deterministic (`argmax`): Hold-only
  - Stochastic sampling: produces Buy/Sell
- Added sidebar policy mode control in Signal Analytics:
  - `Deterministic policy (argmax action)` (default ON)
  - OFF enables sampled diagnostics.

### 5) Analytics layout, P&L, and comparison metrics
- Moved chart section directly below Signal Analytics header.
- Added Total P&L to Performance Summary.
- Added Theoretical Improvement Headroom block:
  - Theoretical Max Accuracy

- Expanded analytics tables:
  - Action Mix includes `count`, `total_pnl`, `avg_trade_edge`
  - Detailed Signal Log includes `reward`, `net_worth`, `cumulative_pnl`
